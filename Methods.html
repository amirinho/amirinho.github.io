What are the data migration methods in cloud computing

To move applications and data to more advantageous environments, gather recommends data and applications as a means to overcome data gravity .firms improve their data management through data migration methods which are as follows:

Types of data migration BIG bang migration

Big bang data migration is when the full transfer is completed within a limited time frame ,live systems experience downtime while data goes through ETL processing and transition to a new database . in this method users experience a blockade in a single point I time where they can no longer use the old data until the new system goes live again Bigbang migrations typically have significant preparation periods and shorter down times, during which the system is unavailable.

TRICKLE MIGRATION

A trickle migration approach can be likened to an agile form to data migration.in contrast the complete data migration process is done in phases ,during this process the old and the new system are run in parallel which eliminates downtime or operational interruptions.in this approach it allows all stakeholders to verify the success of every phase in data migration process submitting a significant sign of progress in this method. In comparison with the Bigbang method this implementation can be complex in design but if it is done right it usually reduces risks rather than adding them
